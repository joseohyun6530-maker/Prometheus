# PRD: COZY — 주문하기 화면

| 항목 | 내용 |
|------|------|
| **문서 버전** | 1.0 |
| **대상 화면** | 주문하기 (Order) |
| **참조 와이어프레임** | `wf-1.png` (주문하기 화면) |
| **관련 화면** | [관리자](./PRD-admin.md) (`wf-2.png`) |
| **구현 경로** | `ui/src/pages/OrderPage.jsx` 및 하위 컴포넌트 |
| **API 베이스** | `GET /api/v1/menus`, `POST /api/v1/orders` |

---

## 1. 개요

### 1.1 목적

방문 고객이 메뉴를 확인하고, 옵션을 선택해 장바구니에 담은 뒤, 한 번에 주문을 제출할 수 있는 **고객용 주문 화면**을 제공한다.

### 1.2 사용자

- 카페 **방문 고객** (키오스크·태블릿·웹 단일 디바이스에서 주문)

### 1.3 앱 내 위치

- 상단 공통 헤더: **COZY** 로고, **주문하기** | **관리자** 탭
- 본 문서는 **주문하기** 탭이 활성화된 상태만 다룬다.
- **관리자** 화면과 주문·재고 데이터가 API로 연동된다.

### 1.4 성공 지표 (참고)

- 메뉴 선택 → 담기 → 주문 완료까지 단계 최소화
- 옵션·수량·금액 계산 오류 없음
- 주문 제출 후 관리자 화면에 주문이 반영됨

## 2. 화면 구조 (Information Architecture)

와이어프레임(`wf-1`) 기준 레이아웃:

```
┌─────────────────────────────────────────────────────────┐
│  COZY                          [주문하기*] [관리자]      │  ← 공통 Header
├─────────────────────────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ...            │
│  │  이미지  │  │  이미지  │  │  이미지  │                 │  ← 메뉴 카드 그리드
│  │  메뉴명  │  │  메뉴명  │  │  메뉴명  │                 │
│  │  가격    │  │  가격    │  │  가격    │                 │
│  │  설명    │  │  설명    │  │  설명    │                 │
│  │  □ 옵션  │  │  □ 옵션  │  │  □ 옵션  │                 │
│  │ [담기]   │  │ [담기]   │  │ [담기]   │                 │
│  └─────────┘  └─────────┘  └─────────┘                 │
├─────────────────────────────────────────────────────────┤
│  장바구니                                                │
│  · 라인 목록 (메뉴명, 옵션, 수량, 금액)                   │
│  · 총 금액                          [주문하기]           │
└─────────────────────────────────────────────────────────┘
```

### 2.1 영역별 역할

| 영역 | 컴포넌트 | 설명 |
|------|-----------|------|
| 헤더 | `Header` | 브랜드·탭 전환 (주문하기 활성 표시) |
| 메뉴 | `OrderPage` > `menu-grid` > `ProductCard` | API 메뉴 목록 표시 |
| 장바구니 | `Cart` | 담은 항목·합계·주문 제출 |

## 3. 와이어프레임 대비 UI 요구사항

### 3.1 헤더

| 요소 | 요구사항 |
|------|----------|
| 로고 | 좌측 **COZY** 텍스트 로고 |
| 주문하기 | 활성 탭: 테두리/강조 스타일 |
| 관리자 | 비활성 탭: 클릭 시 관리자 화면으로 전환 (주문하기 화면은 DOM 유지, CSS로 숨김) |

**비기능**

- 탭 전환 시 **장바구니 상태 유지** (앱 루트 `App`에서 `cart` state 관리)

### 3.2 메뉴 카드 (`ProductCard`)

와이어프레임 1장당 1메뉴. 카드 구성:

| 요소 | 요구사항 | 데이터 소스 |
|------|----------|-------------|
| 이미지 | `image_url` 있으면 표시, 없으면 placeholder(×) | `GET /menus` → `image_url` |
| 메뉴명 | 예: 아메리카노(ICE) | `name` |
| 가격 | 기본가, 원 단위 표시 (예: 4,000원) | `price` |
| 설명 | 1~2줄 요약 | `description` |
| 옵션 | 체크박스, 복수 선택 가능 | `options[]` |
| 옵션 라벨 | `{이름} (+{가격}원)` (0원도 표시) | `name`, `price` |
| 담기 | 클릭 시 장바구니에 반영 | — |

**와이어프레임 예시 메뉴 (참고)**

| 메뉴명 | 기본가 (와이어프레임) |
|--------|----------------------|
| 아메리카노(ICE) | 4,000원 |
| 아메리카노(HOT) | 4,000원 |
| 카페라떼 | 5,000원 |

**구현 확장 (와이어프레임 외)**

- API 시드 기준 **최대 6개 메뉴** 노출 가능 (바닐라라떼, 카라멜 마키아토, 콜드브루 등)
- 재고 추적 메뉴(ICE/HOT/라떼)에 **재고 뱃지** 표시: `정상` / `주의` / `품절` 및 수량
- 품절 시 카드 비활성·**담기 → 품절** 버튼

### 3.3 장바구니 (`Cart`)

| 요소 | 요구사항 |
|------|----------|
| 제목 | **장바구니** |
| 빈 상태 | "담은 메뉴가 없습니다." |
| 라인 표시 | `{메뉴명} ({선택 옵션 목록})` + 수량 + 라인 금액 |
| 수량 | 와이어프레임: `X n` 텍스트 — **구현**: `−` / 수량 / `+` 로 조절 |
| 총 금액 | 모든 라인 `line_total` 합계, **총 금액** 라벨 |
| 주문하기 | 장바구니 비어 있으면 비활성 |

**와이어프레임 장바구니 예시**

| 라인 | 단가 계산 | 라인 금액 |
|------|-----------|-----------|
| 아메리카노(ICE) (샷 추가) × 1 | 4,000 + 500 | 4,500원 |
| 아메리카노(HOT) × 2 | 4,000 × 2 | 8,000원 |
| **합계** | | **12,500원** |

## 4. 기능 요구사항

### 4.1 메뉴 조회

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| O-01 | 진입 시 `GET /api/v1/menus`로 메뉴·옵션·재고 로드 | 필수 |
| O-02 | 로딩 중 "메뉴를 불러오는 중..." 표시 | 필수 |
| O-03 | API 실패 시 상단 에러 메시지 표시 | 필수 |

### 4.2 옵션 선택

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| O-10 | 메뉴 카드별 옵션은 **독립** 체크 상태 (카드마다 `useState`) | 필수 |
| O-11 | 옵션 미선택 시에도 담기 가능 | 필수 |
| O-12 | 동일 메뉴·동일 옵션 조합은 장바구니 **한 라인으로 병합**, 수량 증가 | 필수 |

### 4.3 담기

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| O-20 | **담기** 클릭 시 단가 = 기본가 + 선택 옵션가 합 | 필수 |
| O-21 | 재고 추적 메뉴: 장바구니 내 해당 메뉴 **총 수량**이 재고 이하일 때만 담기 허용 | 필수 |
| O-22 | 품절(재고 0) 시 담기 비활성, 버튼 문구 **품절** | 필수 |
| O-23 | 재고 부족(담기 후 한도 초과) 시 **재고 부족** 표시 | 필수 |

**재고 추적 메뉴 ID**

- `americano-ice`, `americano-hot`, `cafe-latte`

### 4.4 장바구니 수량 변경

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| O-30 | `+` 클릭 시 수량 1 증가, 라인 금액 재계산 | 필수 |
| O-31 | `−` 클릭 시 수량 1 감소, 0이 되면 라인 삭제 | 필수 |
| O-32 | `+` 시 재고 한도 초과면 수량 변경 없음 | 필수 |

### 4.5 주문 제출

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| O-40 | **주문하기** 클릭 시 클라이언트 재고 검증 후 `POST /api/v1/orders` | 필수 |
| O-41 | 요청 body: `{ items: [{ menu_id, quantity, option_ids[] }] }` | 필수 |
| O-42 | 성공 시 장바구니 비우기, "주문이 접수되었습니다." 알림, 메뉴(재고) 갱신 | 필수 |
| O-43 | 실패 시 API `error.message` 알림 | 필수 |
| O-44 | 빈 장바구니에서 주문하기 불가 | 필수 |

## 5. 비즈니스 규칙

### 5.1 가격 계산

```
단가(unit_price) = 메뉴 기본가 + Σ(선택 옵션 가격)
라인 금액(line_total) = 단가 × 수량
총 금액 = Σ(라인 금액)
```

- 금액은 **정수(원)** 만 사용
- 옵션 ID는 장바구니 키 생성 시 **정렬**하여 동일 조합 병합

### 5.2 장바구니 라인 키

```
{menu_id}:{option_id_1,option_id_2,...}  // option_ids 오름차순
```

### 5.3 재고

| 재고 수량 | 상태 라벨 | 담기 |
|-----------|-----------|------|
| 0 | 품절 | 불가 |
| 1~4 | 주의 | 가능 (한도 내) |
| 5+ | 정상 | 가능 |

- 재고 미추적 메뉴(와이어프레임 외 추가 메뉴): 재고 뱃지·품절 규칙 **미적용**, 담기 항상 가능 (서버는 주문 시 재고 차감)

### 5.4 주문 후

- 서버가 `RECEIVED` 상태로 주문 저장 및 재고 차감
- 관리자 화면 **주문 현황**·**대시보드**에 반영 (별도 PRD)

## 6. API 연동

### 6.1 메뉴 목록

```
GET /api/v1/menus
```

**응답 사용 필드 (메뉴 카드)**

| API 필드 | UI 매핑 |
|----------|---------|
| `id` | 내부 식별 |
| `name` | 메뉴명 |
| `description` | 설명 |
| `price` | 기본가 |
| `image_url` | 이미지 URL (`/americano-ice.png` 등) |
| `stock_quantity` | 재고·품절 판단 |
| `options[].id` | 옵션 체크박스 value |
| `options[].name` | 옵션 라벨 |
| `options[].price` | 옵션 추가가 |

### 6.2 주문 생성

```
POST /api/v1/orders
Content-Type: application/json

{
  "items": [
    {
      "menu_id": "americano-ice",
      "quantity": 1,
      "option_ids": ["extra-shot"]
    }
  ]
}
```

**에러 (예시)**

| HTTP | code | UI 처리 |
|------|------|---------|
| 409 | `INSUFFICIENT_STOCK` | 알림으로 메시지 표시 |
| 400 | `INVALID_REQUEST` | 알림 |
| 404 | `MENU_NOT_FOUND` | 알림 |

## 7. 상태·예외 처리

| 상태 | 조건 | UI |
|------|------|-----|
| 로딩 | 최초 메뉴 fetch | "메뉴를 불러오는 중..." |
| 정상 | 메뉴 표시 | 카드 그리드 + 장바구니 |
| API 오류 | fetch 실패 | 상단 빨간 `app-error` 배너 |
| 품절 | `stock_quantity === 0` | 카드 `product-card--soldout` |
| 장바구니 빈 | `cart.length === 0` | 안내 문구, 주문하기 disabled |

## 9. 와이어프레임 vs 현재 구현 차이

| 항목 | 와이어프레임 | 현재 구현 |
|------|-------------|-----------|
| 메뉴 개수 | 3개 | API 기준 최대 6개 |
| 재고 표시 | 없음 | 3종 메뉴에 재고 뱃지 |
| 장바구니 수량 | `X n` 텍스트 | `−` / 숫자 / `+` 컨트롤 |
| 이미지 | placeholder | 3종 PNG, 나머지 placeholder |
| 탭 전환 | — | 장바구니 유지 (페이지 unmount 없음) |

향후 와이어프레임과 **완전 일치**가 필요하면: 메뉴 3종 필터, 재고 뱃지 숨김, 장바구니 수량 표시 단순화 등을 별도 스프린트로 정의한다.

## 10. 인수 조건 (Acceptance Criteria)

- [ ] 주문하기 탭에서 API 메뉴가 카드 형태로 표시된다.
- [ ] 옵션 선택 후 담기 시 장바구니에 올바른 단가·옵션 문구가 반영된다.
- [ ] 동일 메뉴·옵션 재담기 시 수량만 증가한다.
- [ ] 와이어프레임 예시와 같이 샷 추가 시 500원이 반영된다 (4,000 → 4,500).
- [ ] 장바구니 총액이 라인 합계와 일치한다.
- [ ] 주문하기 제출 후 서버에 주문이 생성되고 장바구니가 비워진다.
- [ ] 품절 메뉴는 담을 수 없다.
- [ ] 관리자 탭 갔다가 돌아와도 장바구니가 유지된다.
